# 사용자 가이드

> 최종 업데이트: 2026-05-08

---

## 기본 사용 흐름

1. 시트1 설정 (A2 강조키워드, B2 스타일번호, C2 SEO키워드, D2 템플릿, F열 발행모드)
2. `runGenerateOnly()` 실행 → SEO 글 생성 + Drive 이미지 폴더 자동 생성
3. K열의 Drive 폴더 URL 접속 → 이미지 파일 업로드
4. `runPublishOnly()` 실행 → 사진 매핑 → HTML 변환 → Blogger 발행 → L열 URL 기록

---

## 중복 발행 방지

- **이미 발행된 글(L열 URL 있음)은 `runPublishOnly()` 재실행해도 자동 차단됨**
  - `getPublishControlRow_()`가 L열에 URL이 있는 행을 탐색에서 제외
  - 발행 가능한 행이 없으면 즉시 종료 + 토스트 알림 출력

- **재발행이 필요한 경우**
  1. 시트1 해당 행의 L열 URL 삭제
  2. `runPublishOnly()` 재실행

- **발행 완료 후 `_final_seo.json` 자동 삭제**
  - 발행 성공 시 Drive JSON 폴더의 해당 파일이 자동으로 휴지통 이동
  - JSON 파일 없이 `runPublishOnly()` 실행 시 오류로 차단됨

---

## 이미지 파일명 규칙

- **파일명은 숫자로 시작해야 합니다**
  - `01.jpg` ✅ `02.png` ✅ `1.jpg` ✅
  - `사진1.png` ❌ `photo01.jpg` ❌ (숫자 외 문자로 시작하면 무시됨)

- **확장자는 제한 없음** — jpg, png, gif, webp 등 Drive에서 지원하는 모든 이미지 형식 허용
  - 코드 필터 조건: 파일명이 숫자로 시작하는지(`/^\d+/`)만 확인, 확장자 검사 없음

- **정렬 기준**: 파일명 앞 숫자 오름차순
  - `01` → `[사진1]`, `02` → `[사진2]` 순으로 자동 매핑

- **권장 형식**: `01.jpg`, `02.jpg` ... 두 자리 번호 사용 권장 (10개 이상일 경우 정렬 보장)

---

## 소스파일 중복 처리 방지

- **이미 처리된 소스파일은 `recordProcessedFile()`로 자동 스킵됨**
  - 처리 완료된 파일 ID를 Properties Service에 기록
  - `runGenerateOnly()` 재실행 시 기록된 파일은 건너뜀

- **INPUT 폴더에 기존 파일이 남아있어도 새 파일만 처리됨**
  - 폴더를 정리하지 않아도 중복 글 생성 없음

## 발행모드별 동작

| F열 값 | H열 체크박스 | 동작 |
|--------|-------------|------|
| `자동` | 무관 | Blogger에 즉시 공개(LIVE) 발행 |
| `수동승인` | 미체크 | G열 `승인대기` 기록 후 보류 |
| `수동승인` | 체크 ☑ | Blogger에 임시저장(DRAFT) 발행 |

---

## 예약발행

- 시트1 I열에 날짜시간 입력 시 해당 시간에 예약발행
  - 예: `2026-05-10 10:00`
  - 내부적으로 ISO 8601 형식으로 변환 후 Blogger API `published` 필드에 전달
- I열 비어있으면 즉시 발행

---

## 주요 함수 요약

| 함수 | 설명 |
|------|------|
| `runGenerateOnly()` | 소스파일 전처리 + SEO 글 생성 + Drive 폴더 생성 |
| `runPublishOnly()` | 사진 매핑 + HTML 변환 + Blogger 발행 |
| `dryRunPublish()` | 실제 발행 없이 예약발행 로직 검증 |
| `resetAllProcessedFiles()` | 처리 완료 기록 초기화 (재처리 필요 시) |
