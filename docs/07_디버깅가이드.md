# 07_디버깅가이드

## 개요
디버깅 항목은 4개 순위로 분류하여 순차 적용했다.

---

## 1순위 — 실행 흐름 안정성

### 시트 설정값 누락 검증
- 대상 함수: `validateControlSheet_()`
- 연결: `runGenerateOnly()`, `runPublishOnly()`, `runAll_Auto()`
- 검증 항목:
  - A열: 강조키워드
  - C열: 검색최적화키워드
  - F열: 발행모드
  - J열: 이미지소스
- 누락 시 팝업으로 어떤 열을 채워야 하는지 안내 후 중단

### J열 미설정 묵시 처리 제거
- 기존: J열 비어있으면 직접업로드로 자동 처리
- 변경: J열 비어있으면 검증 단계에서 명시적으로 중단

### 초기화 후 재실행 안내 강화
- 초기화 완료 토스트:
  - `초기화 완료! 모드를 확인하고 새 파일을 업로드한 뒤 다시 실행하세요.`

---

## 2순위 — 이미지 처리

### OpenAI 502/503/429 재시도
- 대상 함수: `generateImageWithOpenAI_()`
- 최대 3회 재시도
- 재시도 간격: 10초 대기
- 그 외 오류는 즉시 중단

### GitHub 업로드 실패 메시지 세분화
- 401: 토큰 인증 실패
- 403: 접근 권한 없음
- 422: 동일 파일명 충돌
- 그 외: 응답 코드 + 원문 포함

### 발행 후 이미지 URL 검증
- 대상 함수: `verifyImageUrl_()`
- image1/image2 URL 각각 GET 요청으로 접근 확인
- G열 상태 세분화:
  - 정상: `발행완료`
  - 경고: `발행완료(검증주의)`
- 경고 시 토스트: `발행은 완료됐지만 이미지 검증에서 경고가 발생했습니다. 실행 로그를 확인하세요.`

---

## 3순위 — 직접업로드 모드

### 파일명 규칙 위반 감지
- 숫자로 시작하지 않는 파일 감지
- 위반 파일 최대 10개까지 팝업으로 표시
- 올바른 형식 안내: `01.jpg`, `02.png`
- 발행 전 즉시 중단

### 사진 부족 상세 안내
- 유효 파일 수(숫자 시작)와 전체 업로드 수를 함께 표시
- 부족 시 팝업으로 안내 후 중단

### 업로드 폴더 접근 검증
- 대상 함수: `openUploadFolder_()`
- O열 URL에서 폴더 ID 파싱
- `DriveApp.getFolderById()` 실제 접근 확인
- URL 형식 오류/폴더 삭제/권한 문제 시 안내 팝업 후 중단

---

## 4순위 — 시트/데이터

### 필수값 검증 강화
- `runGenerateOnly()`, `runPublishOnly()`, `runAll_Auto()` 시작 시 공통 실행
- A/C/F/J 중 하나라도 비어있으면 즉시 중단

### 모드 미설정 처리
- J열 비어있으면 묵시 처리 없이 명시적 오류 안내

---

## 관련 함수 목록

| 함수 | 역할 |
|------|------|
| `validateControlSheet_()` | 시트 필수값 검증 |
| `verifyImageUrl_()` | 이미지 URL 접근 검증 |
| `getManualImageFileSummary_()` | 업로드 파일 요약 |
| `extractDriveFolderIdFromUrl_()` | Drive 폴더 ID 파싱 |
| `generateImageWithOpenAI_()` | OpenAI 이미지 생성 + 재시도 |
| `uploadImageToGitHub_()` | GitHub 업로드 + 오류 세분화 |
