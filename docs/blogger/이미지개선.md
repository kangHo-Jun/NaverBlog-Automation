# 이미지 생성 개선 방안 — 최종

## 1. 현재 문제

```text
업로드 파일 → JavaScript 전처리
  → Gemini 1회 (최신탐색 Y모드)
  → Claude 본문 생성
  → Claude가 이미지 전략까지 함께 판단
  → Gemini 이미지 생성
```

| 원인 | 내용 |
|------|------|
| 입력 부족 | 핵심 기술정보가 짧게 요약되면 이미지 판단 밀도가 떨어질 수 있음 |
| 판단 주체 오류 | 이미지 전략이 글 구조와 분리되면 정합성이 급격히 낮아짐 |
| 역할 미정의 | 이미지1/2의 교육 목적이 없으면 비슷한 일러스트가 반복됨 |
| 최신정보 미연결 | 탐색 결과가 visual strategy에 녹지 않으면 최신성 체감이 약함 |
| 대산 정체성 부족 | 일반 자재 설명에 머물면 현장성, 발주성, 견적 맥락이 사라짐 |

## 2. 개선 목표 구조

```text
업로드 파일 → JavaScript 전처리
  → Gemini 1회 (최신탐색 Y모드) → 결과를 Claude에 전달
  → Claude 1회:
      - 본문 생성
      - 글 유형 1차/2차 자동 판단
      - visual_strategy 설계
      - 이미지 프롬프트 2개 생성
      → final_seo.json 저장
  → image1: Gemini 1회
  → image2: Apps Script SVG 직접 생성
```

## 3. 역할 분리

| 주체 | 역할 |
|------|------|
| JavaScript | 전처리, 시트 제어, JSON 저장, 발행 파이프라인 제어 |
| Gemini 1회 | 최신정보 탐색, 검색 기반 보강 정보 수집 |
| Claude 1회 | 본문 생성, 글 유형 판단, visual strategy 설계, 이미지 프롬프트 작성 |
| Gemini 1회 | image1 생성만 수행 |
| Apps Script SVG | image2 표/매트릭스/체크리스트 직접 생성 |

## 4. 글 유형 판단 — Claude 자동 판단

| 글 유형 | 판단 기준 | 이미지1 역할 | 이미지2 역할 |
|---------|-----------|--------------|--------------|
| 자재비교 | 두 자재 이상 구조/성능 비교 | 재료 구조/적층 차이 단면도 | 용도별/현장별 선택 기준표 |
| 시공실수 | 실수/오류/주의 언급 | 잘못된 시공 흐름도 | 올바른 시공 순서/체크리스트 |
| 성능설명 | 수치/원리/성능 중심 | 성능 원리 구조도 | 조건별(온도/습도/하중) 수치 비교 |
| 구매가이드 | 선택기준/추천 중심 | 자재 선택 결정 흐름도 | 현장 유형별 추천 매트릭스 |
| 대산브랜딩 | 대산 유통/견적/AI 중심 | 기존 유통 vs 대산 유통 구조 비교 | AI 견적/즉시 발주/물류 흐름도 |
| 현장문제해결 | 현장 하자/문제 원인 분석 | 문제 발생 원인 구조도 | 해결 전/후 비교 또는 점검 포인트 |
| 규격수치설명 | 두께/규격/등급 수치 중심 | 규격 치수 표시도 | 수치별 적용 기준 비교표 |
| 비용물류판단 | 비용/발주/납기/물류 중심 | 자재 선택에 따른 비용 구조도 | 발주/납품/현장 손실 비교표 |
| 기준규정설명 | 방화/인증/KS/법규 중심 | 기준/인증 적용 범위 구조도 | 현장별 요구 기준 체크표 |

## 5. Claude 출력 JSON 구조

```json
{
  "content": "# 제목\n부제목\n\n본문...",
  "content_type": {
    "primary": "자재비교",
    "secondary": "시공실수",
    "reason": "판단 근거 1문장"
  },
  "visual_strategy": {
    "image1": {
      "role": "역할",
      "goal": "교육 목표",
      "must_include": ["요소1", "요소2"],
      "avoid": ["금지요소1", "금지요소2"]
    },
    "image2": {
      "role": "역할",
      "goal": "교육 목표",
      "must_include": ["요소1", "요소2"],
      "avoid": ["금지요소1", "금지요소2"]
    }
  },
  "imagen_prompts": [
    {"image_no": 1, "prompt": "..."},
    {"image_no": 2, "prompt": "..."}
  ]
}
```

## 6. 이미지 프롬프트 규칙

- `primary` + `secondary`를 먼저 판단한다.
- 그 다음 `visual_strategy`를 먼저 설계한다.
- `imagen_prompts`는 반드시 `visual_strategy`를 근거로 생성한다.
- `image1.role`은 반드시 `image1_role_map`의 primary 기준값을 그대로 사용한다.
- `image2.role`은 반드시 `image2_role_map`의 primary 기준값을 그대로 사용한다.
- `secondary`는 소재, 현장조건, 비교항목 보강에만 사용한다.
- "현장에서 왜 이 판단이 중요한지"가 보이도록 시각화한다.
- 이미지 내 텍스트는 최소화하고 짧은 라벨만 허용한다.
- 기술 용어는 영어 유지한다. 예: `LVB`, `LVL`, `MPa`, `KS`
- 일반 설명 라벨도 영어로 유지한다.
- 라벨은 2~3단어 이내로 최소화한다.
- 회사명, 브랜드명, 지역명은 금지한다.
- 대산 맥락은 현장 문제, 발주, 견적, 자재 선택 흐름으로만 드러낸다.
- `기준규정설명` 유형이면 반드시 아래 규칙을 추가한다.
  - 법적 확정 표현 금지. 참고용 구조로만 표현.

## 7. image1_role_map 강제 규칙

- 자재비교: 재료 구조/적층 차이 단면도
  - `side by side` 레이아웃 필수
  - 결 방향 표현 유지 필수
- 시공실수: 잘못된 시공 흐름도
- 성능설명: 성능 원리 구조도
- 구매가이드: 자재 선택 결정 흐름도
- 대산브랜딩: 기존 유통 vs 대산 유통 구조 비교
- 현장문제해결: 문제 발생 원인 구조도
- 규격수치설명: 규격 치수 표시도
- 비용물류판단: 자재 선택에 따른 비용 구조도
- 기준규정설명: 기준/인증 적용 범위 구조도

## 8. photo_guides 출력 규칙

- `photo_guides`는 전체를 한글로 출력한다.
- 예:
  - `벽체 리모델링 전후 비교 사진`
  - `시공 도구 및 자재 준비 사진`
  - `실내 공기질 문제 상황 사진`
- 사진 플레이스홀더도 `[사진 X: 한글 설명]` 형식을 사용한다.

## 9. 이미지 텍스트 언어 정책

- 공통 원칙:
  - `minimal English labels only` 표현은 사용하지 않는다.
- 허용:
  - 짧은 한글 라벨: 자재명, 부위명, 조건명, 비교기준, 적합도
  - 예: `일반합판`, `방수석고`, `습기환경`, `긴스팬`, `권장`, `주의`, `적합`
  - 기술 약어 영문 유지: `LVB`, `LVL`, `KS`, `MPa`, `mm`, `T`, `E0`
- 금지:
  - 긴 한글 문장
  - 법규 단정 표현
  - 구조 계산 단정
  - 가격/성능 보장 문구
  - 브랜드 독점 표현
- 이미지 유형별 언어 선택:
  - 선택기준표, 매트릭스, 체크리스트, 비교표, 수치비교표: 후처리용 프롬프트
  - 단면도, 구조도, 원리도, 흐름도: 한글 라벨 허용 프롬프트
  - 단면도/구조도: Korean labels + English abbreviations
  - 선택기준표/매트릭스/체크리스트: 후처리용 무문자 구조 프롬프트
  - 흐름도/원리도: minimal text, icons and arrows preferred, short Korean labels only
  - 수치비교표: 후처리용 무문자 구조 프롬프트 + 숫자 영역 placeholder
  - 법규/인증 설명: 한글 최소화, 단정 표현 절대 금지
- 프롬프트 끝 문장 규칙:
- image2가 표/매트릭스/체크리스트/비교표/수치비교표 유형이면 아래 문장을 붙인다.
- `Create clean table structure with simple icons, checkmarks (✓), triangles (△), and X marks only. No readable text, no Korean, no English labels. Use blank header areas and placeholder blocks where labels will be added later. Flat technical illustration, clean white background, no people, no brand names.`
- image1과 단면도/구조도/원리도/흐름도 유형은 아래 문장을 유지한다.
- `Short readable Korean technical labels only, keep standard abbreviations in English/symbols (LVB, LVL, KS, mm, MPa), no long Korean sentences.`
- 보조 표현 적극 사용:
  - `◎ 권장 / △ 조건부 / × 비권장`
  - 화살표로 흐름 표현
  - 체크마크로 적합도 표시
  - 아이콘으로 조건 구분 (습기, 하중, 비용 등)

## 10. image2 SVG 직접 생성 방식

- image2는 Gemini로 생성하지 않는다.
- `visual_strategy.image2.must_include`를 사용해 Apps Script가 SVG 표를 직접 만든다.
- 기본 방식:
  - 선택기준표/매트릭스: 조건명 1열 + 자재명 2~4열 + `✓ / △ / ×`
  - 체크리스트: 항목 목록 + `✓ / △ / ×`
  - 비교표/수치비교표: 항목명 + 수치 placeholder 셀
- 스타일:
  - 800x600
  - 흰색 배경
  - 헤더 `#2c5f8a`
  - 짝수 행 `#f0f4f8`
  - 폰트 `Noto Sans KR`
  - 기호색: `✓ #2c5f8a / △ #f5a623 / × #e74c3c`
- 결과물은 SVG 그대로 GitHub에 업로드하고 Blogger에 삽입한다.

## 11. 이미지 품질 평가 기준

| 항목 | 기준 |
|------|------|
| 본문 정합성 | 글의 핵심 논점과 직접 연결되는가 |
| 기술 구체성 | 수치, 구조, 비교 기준이 있는가 |
| 현장성 | 시공/발주/자재 선택 상황이 보이는가 |
| 차별성 | 흔한 일러스트가 아닌가 |
| 대산 연결성 | 전문성/유통/견적 맥락이 보이는가 |
| 텍스트 안정성 | 이미지 내 글자가 깨지지 않는가 |

## 12. 구현 우선순위

| 순위 | 작업 |
|------|------|
| 1 | Claude 시스템 프롬프트를 9개 유형 + visual_strategy 중심으로 교체 |
| 2 | final_seo.json에 `content_type`, `visual_strategy`, 객체형 `imagen_prompts` 저장 |
| 3 | runPublishOnly()가 `imagen_prompts[].prompt`를 읽도록 수정 |
| 4 | 이미지 품질 평가 후 필요 시 전략 재생성 루프 검토 |

## 13. image1 자동 검증 규칙

- image1.role이 `image1_role_map`과 다르면 실패
- image1 프롬프트가 `construction site / installation scene / building exterior` 위주인데 단면도/흐름도/구조도 개념이 없으면 실패
- `primary_type=자재비교`인데 image1에 `side by side`가 없으면 실패
- `primary_type=자재비교`인데 image1에 `grain direction` 표현이 없으면 실패
- 실패 시 시트 G열에 `이미지재생성` 기록 후 재생성 시도

## 14. image2 자동 검증 규칙

- `primary_type=자재비교`인데 image2에 `selection guide`가 없으면 실패
- image2 프롬프트에 `matrix / table / checklist / criteria / guide` 계열 표현이 없으면 실패
- image2.must_include 비교 기준이 3개 미만이면 실패
- image2.role이 primary 기준 `image2_role_map`과 다르면 실패
- 실패 시 시트 G열에 `이미지재생성` 기록 후 재생성 시도
