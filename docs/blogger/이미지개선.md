# 이미지 개선

## 1. 현재 이미지 생성 흐름

```text
업로드 파일 (HTML/MHTML)
  → runGenerateOnly()
      → Claude API → 블로그 본문 생성
      → generateImagenPrompts_(content, title, highlightKeywords)
          → Gemini Flash (본문 앞 2000자 + 제목 + 강조키워드 입력)
          → 출력: {"prompt1": "...", "prompt2": "..."}
          → final_seo.json 저장
  → runPublishOnly()
      → generateImageWithGemini_(prompt)
          → gemini-3-pro-image-preview 이미지 생성
          → GitHub raw URL → Blogger 삽입
```

## 2. 문제 원인 분석

| 구분 | 현재 상태 | 문제점 |
|------|-----------|--------|
| 입력 범위 | 본문 앞 2000자만 | 핵심 기술정보가 중반 이후에 있으면 누락 |
| Gemini 지시 | "제품 구조/단면도", "성능 비교" 고정 | 글 내용과 무관하게 동일한 구도 반복 |
| 프롬프트 추상성 | 키워드 수준 묘사 | 구체적 수치/원리/비교대상 없음 |
| 최신정보 미반영 | 검색 탐색 결과가 이미지 프롬프트에 미연결 | 최신정보는 글에만 사용, 이미지엔 미전달 |
| 이미지 역할 미정의 | 이미지1/이미지2 역할이 고정 템플릿 | 글의 핵심 논점에 맞는 역할 배분 없음 |

## 3. 핵심 원인 요약

> Gemini가 "무엇을 그려야 하는지"를 스스로 판단할 만한 정보가 부족하다.
> 키워드 수준의 추상적 지시 → 일반적인 일러스트 생성에 그침.

## 4. 개선 방향 아이디어

### [방향 A] 입력 품질 개선

- 본문 앞 2000자 → 전체 본문에서 "기술적 문장만" 추출 (수치, 원리, 비교 포함 문장)
- 강조키워드 + 최신탐색 결과 요약도 함께 전달

### [방향 B] Gemini 지시 구체화

- 고정 템플릿(단면도/비교) 폐기
- Gemini에게 "이 글에서 독자가 가장 이해하기 어려운 기술 개념 1가지를 선택하고, 그것을 시각적으로 설명하는 일러스트를 묘사하라" 방식으로 전환

### [방향 C] 이미지 역할 분리 재정의

- 이미지1: 원리/구조 설명 (How it works)
- 이미지2: 효과/결과 비교 (Before/After 또는 vs 비교)
- 글의 H2 섹션 중 가장 중요한 2개를 각 이미지에 1:1 매핑

### [방향 D] Claude가 이미지 프롬프트 직접 생성

- Gemini 대신 Claude가 본문 생성 직후 이미지 프롬프트도 함께 출력
- 글 내용을 가장 잘 아는 주체가 직접 묘사 → 정합성 최대화
- Claude API 응답에 `imagen_prompts` 필드를 JSON으로 포함하도록 지시

## 5. 추천 우선순위

| 우선순위 | 방향 | 이유 |
|----------|------|------|
| 1순위 | D (Claude 직접 생성) | 가장 정합성 높음, 추가 API 호출 불필요 |
| 2순위 | B (Gemini 지시 구체화) | 현재 구조 유지하며 즉시 적용 가능 |
| 3순위 | C (역할 재정의) | B와 함께 적용 시 효과 극대화 |
| 4순위 | A (입력 품질) | B,C 적용 후 미흡하면 추가 |
