# 06_프롬프터최적화

## 목표
- 불필요한 장문 지시를 줄이고 출력 품질은 유지한다.
- 본문, 이미지 전략, 안전 검수 항목을 한 번에 안정적으로 받는다.
- 재시도 시에도 같은 JSON 스키마를 유지한다.

## 토큰 절약 규칙
- 반복 규칙은 개별 섹션에 다시 쓰지 말고 공통 블록으로 묶는다.
- 금지 규칙은 유사 항목끼리 통합한다.
- 이미지 규칙은 `image1`, `image2`, `공통 언어 정책` 3구간으로 정리한다.
- 긴 예시는 한 개만 두고 나머지는 키 이름으로 제어한다.
- SEO, 가격, 표 규칙은 별도 블록 이름으로 호출한다.

## 출력 JSON 스키마 고정
- 최종 출력은 JSON 객체 하나만 허용
- Markdown 코드블록 금지
- 필수 키:
  - `content`
  - `content_type`
  - `visual_strategy`
  - `imagen_prompts`
  - `image2_table_data`
  - `fact_safety_check`
- 재시도 프롬프트에서도 키 이름과 자료형을 바꾸지 않는다.

## 공통 규칙 블록화 방향
- `콘텐츠 품질 블록`: 가격, 용어, 비교표, SEO, 법규
- `이미지 전략 블록`: 9개 유형, role_map, 텍스트 정책
- `출력 스키마 블록`: JSON 키와 자료형
- `검수 블록`: `fact_safety_check`, `image2_table_data` 최소 조건

## 유지 원칙
- 프롬프트를 줄이더라도 `image2_table_data`와 `fact_safety_check`는 절대 생략하지 않는다.
- role_map 이름과 유형명은 축약하지 않는다.
- 직접업로드, 자동생성 같은 운영 로직은 프롬프트가 아니라 시스템 문서에서 관리한다.
