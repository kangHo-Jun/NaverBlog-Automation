# 02_글생성로직

## 개요
- `createReconstructedPromptWithTemplate()`가 전처리 데이터, 템플릿, 스타일, 키워드를 묶어 Claude 입력을 만든다.
- 결과는 본문만 받지 않고 발행용 JSON 전체 구조로 고정 출력받는다.
- 저장 시점에 `_final_seo.json`으로 정규화된다.

## Claude 출력 구조
```json
{
  "content": "완성 본문",
  "content_type": {
    "primary": "자재비교",
    "secondary": "구매가이드",
    "reason": "판단 근거 1문장"
  },
  "visual_strategy": {
    "image1": {
      "role": "역할",
      "goal": "교육 목표",
      "must_include": ["요소1", "요소2"],
      "avoid": ["금지요소1", "금지요소2"]
    },
    "image2": {
      "role": "역할",
      "goal": "교육 목표",
      "must_include": ["요소1", "요소2"],
      "avoid": ["금지요소1", "금지요소2"]
    }
  },
  "imagen_prompts": [
    {"image_no": 1, "prompt": "이미지1 영어 프롬프트"},
    {"image_no": 2, "prompt": "이미지2 영어 프롬프트"}
  ],
  "image2_table_data": {
    "type": "svg_table",
    "title": "표 제목",
    "columns": ["열1", "열2", "열3"],
    "rows": [
      {"criteria": "기준명", "values": ["◎", "○", "△"]}
    ],
    "legend": {"◎": "권장", "○": "적합", "△": "조건부"},
    "note": "실제 선택은 제품 등급, 하중 조건, 현장 환경에 따라 달라질 수 있음"
  },
  "fact_safety_check": {
    "contains_exact_price": false,
    "contains_unsourced_percent": false,
    "contains_unsourced_lifespan": false,
    "contains_awkward_seo_phrase": false,
    "comparison_sections_use_tables": true
  }
}
```

## 생성 단계
1. `primary`와 `secondary`를 9개 유형 중에서 판정한다.
2. `content_type.reason`에 근거를 1문장으로 기록한다.
3. `visual_strategy.image1`과 `visual_strategy.image2`를 먼저 설계한다.
4. 그 전략을 바탕으로 `imagen_prompts`를 만든다.
5. image2가 표형이면 `image2_table_data`를 함께 만든다.
6. 본문 안전성 점검 결과를 `fact_safety_check`에 넣는다.

## 9개 유형
- 자재비교
- 시공실수
- 성능설명
- 구매가이드
- 대산브랜딩
- 현장문제해결
- 규격수치설명
- 비용물류판단
- 기준규정설명

## 강제 규칙
- 본문 생성 후 `content_type`, `visual_strategy`, `imagen_prompts`, `image2_table_data`, `fact_safety_check`를 반드시 함께 출력한다.
- 자재, 성능, 용도 비교가 있으면 Markdown 테이블을 반드시 넣는다.
- `image2_table_data.rows`는 최소 4개 이상이어야 한다.
- 제공 자료에 없는 가격, 수명, 성능 수치를 임의 생성하면 안 된다.
- image2는 적용 사례 사진이 아니라 표, 매트릭스, 체크리스트, 비교표, 흐름도 중 하나여야 한다.
