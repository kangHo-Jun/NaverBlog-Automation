# 05_이미지생성로직

## 기본 역할
- `image1`: 글의 핵심 판단 구조를 한 장으로 이해시키는 설명용 이미지
- `image2`: 선택 기준, 비교표, 체크리스트, 매트릭스, 흐름도를 SVG 또는 구조형 이미지로 제공하는 보조 이미지

## 9개 유형 role_map

### image1_role_map
- 자재비교 -> 재료 구조/적층 차이 단면도
- 시공실수 -> 잘못된 시공 흐름도
- 성능설명 -> 성능 원리 구조도
- 구매가이드 -> 자재 선택 결정 흐름도
- 대산브랜딩 -> 기존 유통 vs 대산 유통 구조 비교
- 현장문제해결 -> 문제 발생 원인 구조도
- 규격수치설명 -> 규격 치수 표시도
- 비용물류판단 -> 자재 선택에 따른 비용 구조도
- 기준규정설명 -> 기준/인증 적용 범위 구조도

### image2_role_map
- 자재비교 -> 용도별/현장별 선택 기준표
- 시공실수 -> 올바른 시공 순서/체크리스트
- 성능설명 -> 조건별 수치 비교
- 구매가이드 -> 현장 유형별 추천 매트릭스
- 대산브랜딩 -> AI 견적/즉시 발주/물류 흐름도
- 현장문제해결 -> 해결 전/후 비교 또는 점검 포인트
- 규격수치설명 -> 수치별 적용 기준 비교표
- 비용물류판단 -> 발주/납품/현장 손실 비교표
- 기준규정설명 -> 현장별 요구 기준 체크표

## 자동생성 모드
- `runGenerateOnly()`가 Gemini image1 후보 1장, OpenAI image1 후보 1장, SVG image2 1장을 만든다.
- 결과 URL은 `L`, `M`, `R`열에 저장된다.
- 운영자는 `N열`에서 Gemini/OpenAI 중 최종 image1을 선택한다.
- `runPublishOnly()`는 선택된 image1과 SVG image2를 본문에 삽입한다.

## 직접업로드 모드
- `runGenerateOnly()`는 이미지를 만들지 않는다.
- Drive 폴더 URL과 파일명 가이드를 기록한다.
- `runPublishOnly()`는 숫자 시작 파일명만 읽는다.
- 부족하면 발행을 중단하고, 자동 대체는 사용하지 않는다.

## image2 SVG 정책
- `image2_table_data`가 있으면 그 데이터를 우선 사용한다.
- `columns`, `rows`, `legend`, `note`를 모두 사용해 표 구조를 만든다.
- 데이터가 없을 때만 기존 `must_include` 기반 폴백을 사용한다.

## 검증 및 재생성 규칙
- image1 prompt가 비어 있으면 자동생성 이미지를 만들지 않는다.
- `image2_table_data.columns` 누락 시 재생성 대상이다.
- `image2_table_data.rows`가 4개 미만이면 재생성 대상이다.
- `image1.role`과 `image2.role`이 role_map과 다르면 재시도한다.
- 자동생성 후 URL이 비어 있으면 발행 전에 다시 생성해야 한다.

## 한글 텍스트 정책
- image1의 단면도, 구조도, 흐름도에는 짧은 한글 라벨 허용
- 약어 `LVB`, `LVL`, `KS`, `mm`, `MPa`, `T`는 영문 유지
- image2가 표, 매트릭스, 체크리스트형이면 생성 프롬프트는 무문자 구조 우선
- 긴 한글 문장, 가격 보장 문구, 법규 단정 문구는 금지

## Gemini/OpenAI 2장 비교 선택 방식
- `L열`: Gemini image1 URL
- `M열`: OpenAI image1 URL
- `N열`: 운영자가 선택한 최종 image1
- 발행 시 `processAutoImage_()`가 `N열` 값을 읽어 image1을 결정한다.
