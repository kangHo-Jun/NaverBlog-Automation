# 블로그 스타일 가이드

이 문서는 시트2 스타일, 시트3 템플릿, 현재 HTML 디자인 스타일을 한 번에 이해할 수 있도록 정리한 운영 문서다.

---

## ① 시트2 스타일 번호별 특징

주의:

- 코드에는 "스타일 번호별 설명문"이 하드코딩되어 있지 않다.
- 실제 번호별 성격은 시트2에 입력된 값으로 결정된다.
- 아래 표는 운영용 추천 해석 예시다.

| 번호 예시 | 문체 특징 | 추천 사용 상황 |
|---|---|---|
| 1 | 가장 기본형. 전문성과 친근함 균형 | 일반 제품 소개 |
| 2 | 친근하고 쉬운 설명 위주 | 초보자 대상 정보글 |
| 3 | 비교적 객관적이고 정리형 | 비교형, 체크리스트형 |
| 4 | 경험담과 현장감 강조 | 상담 유도형 글 |
| 5 | 전문 용어와 기술 설명 강화 | B2B, 기술 해설형 |

실제 운영에서는 시트2 각 행의 숫자 조합으로 스타일이 결정된다.

## ② 스타일 수치 항목 설명

### `writing_tone`

- 범위: 1~10
- 의미: 글의 전문성, 격식, 거리감

변화 예시:

- 1~3: 친근하고 블로그형
- 4~6: 읽기 쉬운 전문가형
- 7~8: 전문성과 신뢰감 강화
- 9~10: 보고서형, 격식형

### `sentence_style`

- 의미: 문장 길이와 설명 밀도

변화 예시:

- 낮음: 짧고 빠른 문장
- 중간: 설명형과 간결형 균형
- 높음: 길고 해설적인 문장

### `personal_touch`

- 의미: 개인 경험, 현장감, 주관 표현 비중

변화 예시:

- 낮음: 객관적, 사실 중심
- 중간: 경험을 적절히 섞음
- 높음: "제가 현장에서 보면..." 같은 표현 증가

### `visual_richness`

- 의미: 목록, 불릿, 구조화 사용량

변화 예시:

- 낮음: 연속 문단형
- 높음: 체크리스트, 비교 포인트, 단계 정리 증가

### `narrative_flow`

- 의미: 스토리텔링 강도

변화 예시:

- 낮음: 논리 순서형 설명
- 높음: 상황 → 문제 → 해결 흐름 강화

### `expertise_level`

- 의미: 전문 용어와 기술 설명 수준

변화 예시:

- 낮음: 초보자도 쉬운 표현
- 중간: 전문 지식을 쉽게 설명
- 높음: 기술 용어와 세부 수치 적극 사용

### `avg_sentence_len`

- 의미: 평균 문장 길이 목표

변화 예시:

- 20~30: 짧고 템포 빠름
- 35~45: 가장 무난함
- 50 이상: 전문 해설형

### 수치 조합 예시

- 쉬운 설명형
  - `writing_tone 5`, `sentence_style 4`, `expertise_level 4`
- 균형형
  - `writing_tone 6`, `sentence_style 6`, `personal_touch 6`
- 전문가형
  - `writing_tone 8`, `sentence_style 8`, `expertise_level 8`

## ③ 시트3 템플릿별 설명

코드에서 읽는 시트3 항목:

- `template_name`
- `writing_style`
- `content_structure`
- `key_focus_areas`
- `photo_guide_type`
- `tone_description`
- `cta_style`
- `seo_strategy`
- `sample_prompt`

### 문제해결형

- 구조: 문제 제시 → 원인 설명 → 해결 방법 → 추천 자재 → 문의 유도
- 특징: 공감형 오프닝과 해결 중심 전개
- 추천 상황: 결로, 곰팡이, 단열, 소음 등 불편 해결형 콘텐츠

### 제품소개형

- 구조: 제품 필요성 → 핵심 특징 → 차별점 → 적용 사례 → CTA
- 특징: 장점과 선택 이유를 분명히 보여줌
- 추천 상황: 특정 자재나 제품군 소개

### 정보제공형

- 구조: 개념 설명 → 특징 정리 → 장단점 → 활용 상황 → 선택 팁
- 특징: 검색형 유입에 강함
- 추천 상황: 자재 차이, 공법 설명, 입문형 정보글

### 비교형

- 구조: 비교 대상 소개 → 차이점 → 장단점 → 추천 기준 → 결론
- 특징: 표, 수치, 비교 이미지와 궁합이 좋음
- 추천 상황: 기존 방식 vs 개선 방식, A 자재 vs B 자재

### 트렌드형

- 구조: 최근 변화 → 핵심 포인트 → 현장 적용 → 전망 → CTA
- 특징: 시기성, 검색성, 제목 클릭률이 좋음
- 추천 상황: 연도별 트렌드, 최신 시공 흐름

## ④ HTML 디자인 스타일

현재 색상 테마:

- 메인 포인트 컬러: `#2c5f8a`

현재 적용 위치:

- H2 상단 구분선
- 카드 숫자 강조
- CTA 및 링크 포인트
- 인용구/하이라이트 강조선

### H2 구분선

역할:

- 큰 섹션 전환을 눈에 띄게 보여줌

현재 스타일:

```html
<h2 style="font-size:1.5em; font-weight:500; margin-top:2em; padding-top:1em; border-top:3px solid #2c5f8a; color:#1a1a1a;">
```

수정 방법:

- 선 두께: `3px`
- 선 색상: `#2c5f8a`
- 여백: `margin-top`, `padding-top`

### 이미지 풀와이드 + 캡션

역할:

- 본문 안에서 이미지를 크게 보여주고 설명을 아래 캡션으로 표시

현재 스타일:

```html
<div style="width:100%; margin:2em 0;">
  <img ... style="width:100%; height:auto; display:block; border-radius:8px;">
  <p style="text-align:center; color:#888; font-size:0.85em; margin-top:0.5em;">...</p>
</div>
```

수정 방법:

- `border-radius`를 줄이면 더 딱딱한 기술 문서 느낌
- `font-size`를 줄이면 캡션 존재감 감소

### 하이라이트 박스

역할:

- `**굵은텍스트:**`로 시작하는 핵심 설명 블록 강조

현재 스타일:

```html
<div style="background:#f0f6ff; border-left:4px solid #2c5f8a; padding:1rem 1.5rem; margin:1.5rem 0; border-radius:0 8px 8px 0;">
```

수정 방법:

- 배경색: `#f0f6ff`
- 강조선 색상: `#2c5f8a`

### 수치 카드

역할:

- `숫자%` 패턴 3개 이상이 연속되면 카드형 그리드로 강조

현재 스타일:

```html
<div style="display:grid; grid-template-columns:repeat(3,1fr); gap:12px; margin:1.5rem 0;">
```

수정 방법:

- 열 개수: `repeat(3,1fr)`
- 카드 배경: `#f5f7fa`
- 숫자 강조 색상: `#2c5f8a`

### 인용구 박스

역할:

- 큰따옴표 문장을 전문가 코멘트처럼 강조

현재 스타일:

```html
<div style="border-left:3px solid #2c5f8a; padding:1rem 1.5rem; margin:1.5rem 0; font-style:italic; color:#555; font-size:1em; line-height:1.8;">
```

수정 방법:

- 선 두께와 색으로 권위감 조절
- `font-style:italic` 제거 시 더 딱딱한 느낌

### CTA 섹션

역할:

- 본문 H2 기준 중간 지점에 자동 삽입
- 첫 SEO 키워드를 자재명처럼 사용

현재 핵심 색상:

- 배경: `#1a3a5c`
- 버튼 텍스트 포인트: 흰색 / 진한 남색

수정 방법:

- CTA 문구를 바꾸려면 `buildCtaHtml()` 내부 문장을 수정
- 링크를 바꾸려면 `https://daesan.ai` 수정

### 회사 서명

역할:

- 글 끝에 신뢰용 서명 고정

현재 스타일:

```html
<div style="border-top:1px solid #ddd; margin-top:3em; padding-top:1.5em; color:#666; font-size:0.9em; line-height:1.8;">
```

수정 방법:

- 회사명 문구
- 소개 문장
- 공식 사이트 링크

## ⑤ 초보자 추천 설정값

가장 무난한 조합:

- 시트2
  - `writing_tone = 6`
  - `sentence_style = 6`
  - `personal_touch = 6`
  - `visual_richness = 5`
  - `narrative_flow = 5`
  - `expertise_level = 6`
  - `avg_sentence_len = 40~45`

- 시트3
  - `문제해결형` 또는 `정보제공형`

추천 이유:

- 너무 딱딱하지 않음
- 검색형 유입과 상담형 글 모두 무난
- 구조와 문체가 과하게 튀지 않음

### 수정 시 체크리스트

1. 글이 너무 딱딱하면 `writing_tone`, `expertise_level`을 함께 낮춘다.
2. 글이 너무 장황하면 `sentence_style`, `avg_sentence_len`을 낮춘다.
3. 글이 심심하면 `personal_touch`, `narrative_flow`를 올린다.
4. 목록형 정리가 필요하면 `visual_richness`를 올린다.
5. 글 구조가 안 맞으면 시트2보다 먼저 시트3 템플릿을 바꿔본다.
