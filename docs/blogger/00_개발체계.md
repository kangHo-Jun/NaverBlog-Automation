# 00_개발체계

## 목적
- 현재 Apps Script 블로그 자동화의 전체 파이프라인을 한 문서에서 관리한다.
- 글 생성, 이미지 처리, 발행, 문서 기준의 책임 구간을 분리한다.
- 운영자가 수정해도 되는 영역과 절대 건드리면 안 되는 영역을 구분한다.

## 최신 전체 파이프라인
1. 제어 시트에서 키워드, 템플릿, 스타일, 발행모드, 이미지소스를 설정한다.
2. `runGenerateOnly()`가 원본 자료를 읽고 Claude용 재구성 프롬프트를 만든다.
3. Claude 결과를 `_final_seo.json`으로 저장한다.
4. `_final_seo.json`에는 본문, 유형 판정, 이미지 전략, 이미지 프롬프트, `image2_table_data`, `fact_safety_check`가 함께 저장된다.
5. 이미지소스가 `자동생성`이면 `runGenerateOnly()` 단계에서 Gemini image1, OpenAI image1, SVG image2 후보를 먼저 만든다.
6. 이미지소스가 `직접업로드`이면 `runGenerateOnly()` 단계에서 이미지 생성 없이 Drive 폴더와 업로드 가이드만 시트에 기록한다.
7. `runPublishOnly()`가 최신 `_final_seo.json`을 읽고 이미지 모드별 후속 처리를 수행한다.
8. 본문은 `convertToBloggerHTML()`에서 HTML로 바뀌며 Markdown 비교표는 HTML 테이블로 변환된다.
9. Blogger 발행 API 호출 후 상태, 발행시간, URL이 시트에 기록된다.

## 모드별 흐름

### 자동생성
1. `J2=자동생성`
2. `runGenerateOnly()`
3. `_final_seo.json` 생성
4. Gemini image1 생성 후 `L열` 저장
5. OpenAI image1 생성 후 `M열` 저장
6. SVG image2 생성 후 `R열` 저장
7. 운영자가 `N열(image1_선택)`에서 Gemini/OpenAI 중 하나를 선택
8. `runPublishOnly()`가 선택된 image1과 image2 SVG를 반영해 발행

### 직접업로드
1. `J2=직접업로드`
2. `runGenerateOnly()`
3. `_final_seo.json` 생성
4. Drive 업로드 폴더 생성
5. `O/P/Q열`에 폴더 URL, 필요 사진 수, 파일명 가이드 기록
6. 운영자가 Drive 폴더에 `01.jpg`, `02.jpg` 형식으로 업로드
7. `runPublishOnly()`가 업로드 수를 다시 확인
8. 부족하면 발행 중단, 충분하면 숫자 순서대로 본문 플레이스홀더에 매핑

## 문서 역할
- `00_개발체계.md`: 전체 구조, 책임 구간, 금지 영역
- `01_시스템가이드.md`: 실운영 순서, 열 구조, JSON 구조, 검증 기준
- `02_글생성로직.md`: Claude 출력 규칙과 본문/JSON 생성 로직
- `03_스타일가이드.md`: 문체, 문단, 제목, 서술 방식
- `04_콘텐츠품질.md`: 가격, 용어, 표, SEO, 법규, 자동 검수 기준
- `05_이미지생성로직.md`: image1/image2 역할, 9개 유형, 검증/재생성 규칙
- `06_프롬프터최적화.md`: 토큰 절약, JSON 스키마 고정, 공통 규칙 블록화
- `blogger연결.md`: Blogger 연결 및 배포 계정 설정 참고

## 절대 수정 금지 영역
- `content_type.primary`의 9개 유형 이름
- `image1_role_map`, `image2_role_map`의 유형별 역할 매핑 기준
- `image2_table_data`와 `fact_safety_check` JSON 키 이름
- 제어 시트 A~R 열 인덱스 체계
- 직접업로드 모드에서 Unsplash 폴백 금지 원칙
- 가격 표현 제한, 용어 현지화, 비교표 강제, SEO 자연어화 규칙의 키 이름

## 수정 가능 영역
- 템플릿 문구
- 스타일 문구
- CTA 가이드
- SVG 시각 스타일
- 시트 헤더 색상과 안내 메시지
