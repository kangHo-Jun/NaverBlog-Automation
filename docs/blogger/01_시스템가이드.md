# 01_시스템가이드

## 초보자용 한눈에 보기

### 전체 과정
```text
원본 파일 업로드
    ↓
시트 설정 입력
  - 글 생성 설정
  - 이미지 생성 방식 선택
  - 발행 방식 선택
    ↓
runGenerateOnly() 실행
    ↓
자동생성 모드인가?
  ├─ 예: SVG image2 생성 + R열 기록
  │      ↓
  │    🤖 OpenAI 이미지 생성 실행
  │      ↓
  │    image1 생성 + M열 기록
  └─ 아니오: 업로드 폴더 생성 + 사진 가이드 기록
    ↓
직접업로드 모드면 Drive 폴더에 사진 업로드
    ↓
runPublishOnly() 실행
    ↓
발행모드 확인
  ├─ 자동: 공개 게시
  └─ 수동승인: Blogger 임시저장(DRAFT)
    ↓
시트에서 결과 확인
  - 상태
  - Blogger URL
  - 임시저장 여부
```

### 가장 쉬운 사용 순서
1. 원본 파일을 Drive 입력 폴더에 올린다.
2. 시트1에서 키워드, 스타일, 템플릿, 발행모드, 이미지소스를 입력한다.
3. 메뉴에서 이미지 모드를 먼저 선택한다.
4. `runGenerateOnly()`를 실행한다.
5. 자동생성 모드면 `🤖 OpenAI 이미지 생성`을 실행해 `M열` URL을 만든다.
6. 직접업로드 모드면 업로드 폴더에 사진을 올린다.
7. `runPublishOnly()`를 실행한다.
8. 시트1의 `G/H/I`를 보고 발행 결과를 확인한다.

## 업로드 방법

### 업로드 가능한 파일 종류
- `.html`
- `.txt`
- `.pdf`

### 업로드 장소
- Google Drive 입력 폴더
- 코드 기준 입력 폴더 ID: `1J_wn9JIilhkyfOBvxkEB1C5B0f5LzNj8`

### 업로드 방법
1. Google Drive에서 입력 폴더를 연다.
2. 원본 파일을 그대로 업로드한다.
3. 한 번에 여러 파일을 올릴 수 있지만, 초보자는 1개씩 처리하는 편이 안전하다.
4. PDF는 OCR로 텍스트를 추출하므로, 스캔 품질이 너무 낮으면 결과가 흔들릴 수 있다.

## 설정 방법

### 시트1에서 사용자가 입력하는 값
- `A2`: 강조키워드
  - 글에서 꼭 강조할 문구
- `B2`: 스타일
  - 시트2 기준 스타일 번호 또는 설정값
- `C2`: 검색최적화키워드
  - 검색 유입용 키워드
- `D2`: 템플릿
  - 글 구조 템플릿
- `E2`: 최신반영여부
  - 최신 검색 보강 사용 여부
- `F2`: 발행모드
  - `자동` 또는 `수동승인`
- `J2`: 이미지소스
  - `자동생성` 또는 `직접업로드`

### 이미지 모드 선택 방법
- 메뉴 `대산 블로그 > 📸 자동생성 모드`
  - `runGenerateOnly()`에서 SVG image2를 만들고, `🤖 OpenAI 이미지 생성`에서 image1을 별도로 만든다.
- 메뉴 `대산 블로그 > 📁 직접업로드 모드`
  - 이미지는 직접 준비해서 Drive 폴더에 업로드

### 발행모드 선택 방법
- `F2=자동`
  - `runPublishOnly()` 실행 시 공개 게시
- `F2=수동승인`
  - `runPublishOnly()` 실행 시 Blogger 임시저장(DRAFT)

## 결과 확인 방법

### 시트에서 확인하는 위치
- `G열`: 현재 상태
- `H열`: 발행시간
- `I열`: Blogger URL

### 자동생성 모드 결과 확인
- `M열`: image1 URL
- `R열`: image2 SVG URL

### 직접업로드 모드 결과 확인
- `O열`: 사진 업로드 폴더 URL
- `P열`: 필요한 사진 수
- `Q열`: 파일명 가이드

### 블로그 주소 확인
- 공개 게시가 완료되면 `I열`에 Blogger 글 주소가 기록된다.
- 이 주소를 클릭하면 실제 게시된 글을 바로 확인할 수 있다.

### 임시보관(임시게시) 확인
- `F2=수동승인`이면 `runPublishOnly()` 실행 결과가 Blogger `DRAFT`로 저장된다.
- 이 경우에도 `I열`에 URL이 기록될 수 있지만, 공개 게시가 아니라 Blogger 편집기에서 보이는 임시저장 글이다.
- 실제 공개는 Blogger 관리자 화면에서 최종 확인 후 직접 게시하면 된다.

## 운영 순서
1. A~J 열의 기본 입력값을 확인한다.
2. 메뉴에서 `📸 자동생성 모드` 또는 `📁 직접업로드 모드`를 선택한다.
3. `runGenerateOnly()`를 실행한다.
4. 자동생성 모드면 `R열` SVG URL을 확인한 뒤 `🤖 OpenAI 이미지 생성`을 실행한다.
5. 직접업로드 모드면 `O/P/Q` 안내에 따라 Drive 폴더에 사진을 올린다.
6. 자동생성 모드에서는 `M열 image1_url`이 채워졌는지 확인한다.
7. `runPublishOnly()`를 실행한다.
8. `G/H/I`에서 상태, 발행시간, Blogger URL을 확인한다.

## 모드별 운영 흐름

### 자동생성
- `J열`: `자동생성`
- `runGenerateOnly()`가 image2 SVG 1장을 만든다.
- `🤖 OpenAI 이미지 생성`이 image1을 별도로 만든다.
- `M열`: image1 URL
- `R열`: image2 SVG URL

### 직접업로드
- `J열`: `직접업로드`
- `runGenerateOnly()`는 이미지를 만들지 않는다.
- `O열`: 업로드 폴더 URL
- `P열`: 필요한 사진 수
- `Q열`: 파일명 가이드와 사진 설명
- `runPublishOnly()`는 업로드 수가 부족하면 발행을 중단한다.

## 검증 기준
- `runGenerateOnly()` 후 최신 `_final_seo.json`이 반드시 생성되어야 한다.
- `image2_table_data.columns`, `image2_table_data.rows`가 존재해야 한다.
- `rows`는 최소 4개 이상이어야 한다.
- `fact_safety_check` 5개 boolean 필드가 모두 채워져야 한다.
- 자동생성 모드에서는 `L/M/R` URL이 채워져야 한다.
- 직접업로드 모드에서는 `O/P/Q`가 채워져야 한다.
- 직접업로드 모드 발행 전 Drive 폴더의 숫자 시작 파일 수가 본문 플레이스홀더 수 이상이어야 한다.
