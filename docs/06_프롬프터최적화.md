# 06_프롬프터최적화

## 핵심 함수
- `createV7HTMLPrompt(preprocessData, seoKeywords, highlightKeywords, templateData, styleData, geminiContext)`

## 구조
- `system`
  - 역할
  - 시점 원칙
  - 수치/사례 원칙
  - 브랜드 노출 원칙
  - AI 티 방지 원칙
  - material-cautions 동적 주입
  - 시트2 스타일 지시
  - 시트3 템플릿 지시
  - 고정 HTML 출력 템플릿
  - 작성 원칙
- `user`
  - SEO 키워드
  - 강조 키워드
  - Gemini 통합 컨텍스트
  - 제목/섹션/차별화/표 생성 지시

## system 프롬프트의 역할
- 문체와 금지 규칙을 강제
- HTML 구조를 사실상 스켈레톤 수준으로 고정
- CTA, FAQ, 서명블록 위치를 하단 기준으로 통제
- 브랜드/AI티/시공주의사항 같은 운영 정책을 통합

## user 프롬프트의 역할
- 이번 글에만 해당하는 입력값 전달
- SEO 키워드와 강조 키워드 전달
- Gemini가 정리한 요약/구조/팩트/SEO보완 전달
- 섹션 수와 템플릿별 구조 제약 전달

## 최적화 원칙
- 반복 규칙은 `system`에 모은다
- 실행마다 바뀌는 값만 `user`에 넣는다
- 스타일/템플릿은 개별 필드를 문자열 블록으로 정리한다
- HTML 코드만 출력하도록 강하게 제한한다

## 현재 고정 출력 원칙
- 표 기반 `<table>` 레이아웃
- 제목/부제목/섹션/특징목록/사진박스/TIP박스
- CTA박스는 FAQ 바로 앞
- FAQ는 CTA 다음
- 서명블록은 문서 맨 마지막

## 주의점
- `createV7HTMLPrompt()`는 품질뿐 아니라 레이아웃 위치까지 좌우한다
- 위치 지시가 약하면 Claude가 CTA나 서명을 상단에 올릴 수 있다
- 따라서 고정 요소는 문구보다 위치 규칙을 먼저 강제해야 한다
