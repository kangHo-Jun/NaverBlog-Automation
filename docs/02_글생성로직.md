# 02_글생성로직

## 메인 파이프라인
```text
runV7HTMLPipeline() [6160]
  → STEP_A_preprocessFiles() [2383]
  → STEP_B_geminiAnalysis() [5818]
  → processNextSEOFile_V7HTML() [5700]
    → createV7HTMLPrompt() [5506]
  → STEP_D2_SaveAsHTML() [5633]
```

## 단계별 설명

### 1. `runV7HTMLPipeline()` [6160]
- V7 HTML 기준 전체 진입점
- 시작 시 JSON 출력 폴더에서 기존 `_preprocess.json`, `_final_seo.json`을 휴지통 처리
- 이후 4단계를 순서대로 실행

### 2. `STEP_A_preprocessFiles()` [2383]
- 내부에서 `preprocessFilesToJson()` 호출
- 입력 폴더에서 새 파일을 읽음
- HTML, TXT, PDF 성격을 구분해 `_preprocess.json` 생성

### 3. `STEP_B_geminiAnalysis()` [5818]
- Gemini로 원본을 한 번 더 요약/구조화
- 핵심 출력:
  - `summary`
  - `outline`
  - `key_facts`
  - `seo_addition`

### 4. `processNextSEOFile_V7HTML()` [5700]
- 현재 V7 글 생성 핵심 함수
- 시트1/시트2/시트3 값을 읽어 최종 프롬프트 구성
- `createV7HTMLPrompt()` [5506]로 `system`, `user` 프롬프트 생성
- Claude 호출 후 HTML만 추출
- `_final_seo.json` 저장

### 5. `STEP_D2_SaveAsHTML()` [5633]
- `_final_seo.json`의 `content`를 읽어 최종 `.html` 파일 생성
- 결과 출력 폴더에 `baseName_최종본_MMdd_HHmm.html` 형식으로 저장

## 산출물

### `_preprocess.json`
- 원본 정리용 중간 파일
- 주요 필드:
  - `fulltext`
  - `content_outline`
  - `reference_snippets`
  - `file_type`

### `_final_seo.json`
- Claude 결과 저장용 파일
- 현재 V7 기준 핵심 필드:
  - `generated_at`
  - `baseName`
  - `file_type`
  - `seo_keywords`
  - `highlight_keywords`
  - `template_data`
  - `content`
  - `model`

## 구형 파이프라인과의 차이
- `runFullPipelineOneByOne()` [2398]와 `processNextSEOFile()` [2155]는 구형 흐름
- 현재 권장 문서 기준은 `runV7HTMLPipeline()` 경로
- `STEP_D1_Simple_SEO_Docs()` [2662]는 Google Docs 기반 구형 산출 경로

## 점검 순서
1. 입력 파일 존재 여부
2. `_preprocess.json`
3. `_final_seo.json`
4. 최종 `.html`
