# 00_개발체계

## 개요
- 프로젝트명: `NaverBlog-Automation`
- 목적: 원본 자료를 전처리하고, Claude/Gemini 기반으로 네이버 블로그용 HTML 본문을 생성한 뒤, 결과 HTML 파일을 Drive에 저장하는 Apps Script 자동화 시스템
- 현재 권장 메인 진입점: `runV7HTMLPipeline()`
- 코드 위치: `src/Code.gs`
- Apps Script 배포 대상: `.clasp.json`의 `scriptId`

## 핵심 파일 구조
- `src/Code.gs`
  - 전체 로직이 들어 있는 단일 Apps Script 소스
- `src/appsscript.json`
  - Apps Script 매니페스트
- `.clasp.json`
  - 로컬 코드와 Apps Script 프로젝트를 연결하는 배포 설정
- `docs/`
  - 운영/개발 문서
- `Naver/`
  - 템플릿/보조 설정 자료

## Drive 폴더 구조
- 입력 폴더: `CONFIG.INPUT_FOLDER_ID`
  - 원본 HTML/TXT/PDF 업로드 위치
- 처리완료 원본 폴더: `CONFIG.PROCESSED_SOURCE_FOLDER_ID`
  - 전처리 후 원본 파일 이동 위치
- 스타일 분석 폴더: `CONFIG.STYLE_ANALYSIS_FOLDER_ID`
  - 과거 분석용 보조 폴더
- JSON 출력 폴더: `CONFIG.JSON_OUTPUT_FOLDER_ID`
  - `_preprocess.json`, `_final_seo.json` 저장 위치
- 결과 출력 폴더: `CONFIG.DOCS_OUTPUT_FOLDER_ID`
  - 최종 `.html` 저장 위치

## 실행 체계
- 전처리: `STEP_A_preprocessFiles()`
- 보강 분석: `STEP_B_geminiAnalysis()`
- HTML 생성: `processNextSEOFile_V7HTML()`
- 파일 저장: `STEP_D2_SaveAsHTML()`
- 일괄 실행: `runV7HTMLPipeline()`

## 권장 운영 방식
1. Drive 입력 폴더에 원본 파일 업로드
2. 시트1에 A~D 및 운영값 입력
3. Apps Script에서 `runV7HTMLPipeline()` 실행
4. JSON 출력 폴더와 결과 출력 폴더 확인

## Git / clasp 배포 체계
- 코드 수정: 로컬 `src/Code.gs`
- 버전 관리: `git add`, `git commit`
- Apps Script 반영: `clasp push`
- 주의:
  - `.clasp.json`이 올바른 Naver Apps Script `scriptId`를 가리키는지 확인
  - 같은 로컬 코드라도 `.clasp.json`이 바뀌면 다른 프로젝트로 배포될 수 있음

## 현재 구조상 주의점
- 메뉴(`onOpen`)에는 과거 경로명이 일부 남아 있을 수 있음
- 현재 문서 기준의 주력 파이프라인은 `runV7HTMLPipeline()`이다
- 구형 Blogger 발행 함수는 대부분 제거되었고, 네이버용 HTML 생성 흐름에 집중해야 한다
