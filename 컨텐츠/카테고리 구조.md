| 설계·사양 | 사양서, VE, 복합 성능, 프로젝트별 사양 |
| 시공·하자 | 스터드, 보드 시공, 조인트, 관통부, 하자 |
| 프로젝트 인사이트 | 주거, 오피스, 병원·학교, 산업시설 |
| 기술 인사이트 | 해외 기준, 친환경, AI 문서화, 용어집 |

이 구조의 장점은 첫째, 사이트 주제가 좁고 선명해서 Google이 “이 사이트는 무엇을 다루는가”를 이해하기 쉽고, 둘째, 생성형 AI가 질문별로 가장 적합한 문서군을 찾기 좋다는 점입니다. 특히 기술 블로그는 사이트 전반의 주제 일관성이 중요합니다. [Google Search Central](https://developers.google.com/search/blog/2022/08/helpful-content-update) [arXiv](https://arxiv.org/html/2311.09735v3)

## 4) SEO 관점에서 우선순위가 높은 카테고리

처음부터 모든 카테고리를 동일한 강도로 운영할 필요는 없습니다. 검색 및 인용 관점에서 초기 우선순위는 아래가 좋습니다.

### 1순위
- 성능 해설
- 문서 해석
- 시공·하자

### 2순위
- 적용 부위
- 설계·사양

### 3순위
- 프로젝트 인사이트
- 기술 인사이트

이유는 업계 기술 자료의 권위 중심축이 **내화/차음 시스템**, **시험성적서/인정서**, **성능 입증 문서**에 있기 때문입니다. 업계 참고 문서도 이러한 카테고리를 중심으로 방대한 자료를 제공하고 있습니다. [gypsum.org](https://gypsum.org/2019/04/ga-600-2018-fire-resistance-and-sound-control-design-manual/) [Knauf](https://knauf.com/ko-KR/tools/download-center)

## 5) GEO 관점에서 특히 강한 카테고리

생성형 AI가 잘 참고하는 글은 대체로 **정의가 명확하고, 조건이 분리되어 있고, 문서 근거가 있는 글**입니다. 그래서 아래 4개 카테고리가 GEO 친화도가 특히 높습니다.

| GEO 우선 카테고리 | 이유 |
|---|---|
| 문서 해석 | 인정서·시험성적서·승인서류처럼 출처가 명확함 |
| 성능 해설 | 내화·차음·내수처럼 질문이 구조화되기 쉬움 |
| 시공·하자 | “왜 문제 생기나”형 질문에 직접 답하기 좋음 |
| 용어집·기준집 | 짧고 명확한 정의문이 AI 답변에 인용되기 쉬움 |

GEO 연구는 **인용, 통계, 관련 출처의 직접 연결**이 가시성을 높일 수 있음을 보여줍니다. 따라서 이 카테고리들에서는 특히 “한 줄 정의”, “비교표”, “공식 문서 출처”, “적용 조건” 블록을 표준화하는 것이 좋습니다. [arXiv](https://arxiv.org/html/2311.09735v3)

## 7) 카테고리별 “허브 글”도 함께 설계해야 합니다

각 대분류는 글이 쌓이기 전에 **허브 페이지**부터 만드는 것이 좋습니다. 예를 들면 아래와 같습니다.

| 대분류 | 허브 글 제목 예시 |
|---|---|
| 성능 해설 | 석고보드 성능 해설 가이드: 내화·차음·내수 핵심 정리 |
| 적용 부위 | 적용 부위별 석고보드 선택 기준: 벽체·천장·샤프트월 |
| 문서 해석 | 석고보드 기술 문서 읽는 법: 인정서·성적서·승인서류 |
| 설계·사양 | 설계자와 공무를 위한 석고보드 사양 검토 가이드 |
| 시공·하자 | 현장에서 자주 발생하는 석고보드 하자와 예방 포인트 |
| 프로젝트 인사이트 | 프로젝트 유형별 석고보드 시스템 적용 인사이트 |
| 기술 인사이트 | 석고보드 기술·기준·지속가능성 트렌드 아카이브 |

이 허브 글은 내부 링크의 중심축이 되며, 검색엔진에는 카테고리의 주제를 설명하고, 생성형 AI에는 사이트의 지식 구조를 보여주는 역할을 합니다. [Google Search Central](https://developers.google.com/search/blog/2022/08/helpful-content-update)

## 8) 대산형 블로그에서 빼는 것이 좋은 카테고리

전문가 대상이라면 아래 카테고리는 메인 축으로 두지 않는 편이 좋습니다.

- “석고보드란 무엇인가” 같은 너무 기초적인 소비자형 입문글
- 단순 제품 카탈로그 재작성형 글
- “추천 TOP 5”식 상업형 리스트
- 트렌드만 좇는 잡다한 건축자재 일반론
- 근거 없이 경쟁사를 비교하는 글

이런 유형은 Google이 경계하는 **검색엔진 우선형/저부가가치형 콘텐츠**로 보일 수 있고, 전문가 독자에게도 신뢰를 주기 어렵습니다. [Google Search Central](https://developers.google.com/search/blog/2022/08/helpful-content-update)

## 9) 구조화 데이터 관점의 카테고리 운영 팁

카테고리 페이지는 목록형 허브로 운영하고, 실제 글은 `Article` 또는 `BlogPosting` 중심으로 구조화하는 것이 좋습니다. 작성자 정보, 작성일, 수정일, 퍼블리셔, 대표 이미지, 작성자 전문 프로필 링크를 명확히 연결하면 전문성 시그널을 강화하는 데 유리합니다. 반면 FAQPage는 적용 범위가 제한적이므로 핵심 전략으로 두기보다는, FAQ 섹션은 본문 안에 넣되 전체 구조는 Article 중심으로 설계하는 편이 안전합니다. [Google Search Central](https://developers.google.com/search/docs/appearance/structured-data/article) [Google Search Central](https://developers.google.com/search/docs/appearance/structured-data/faqpage)

## 10) 최종 추천 구조

가장 실무적으로 정리하면, 대산 블로그는 아래 7축으로 가는 것이 좋습니다.

1. **성능 해설**
2. **적용 부위**
3. **문서 해석**
4. **설계·사양**
5. **시공·하자**
6. **프로젝트 인사이트**
7. **기술 인사이트**

이렇게 가면 SEO에서는 **니치 검색 의도**를 잡을 수 있고, GEO에서는 **문서 근거 기반의 인용형 콘텐츠**를 만들 수 있습니다. 무엇보다 대산이 지향하는 AI/혁신 이미지와도 잘 맞습니다. [회사소개서](https://www.genspark.ai/api/files/s/EXgIlmMO)
